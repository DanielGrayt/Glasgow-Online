Bug reports and feature requests

These guidelines explain how to write a good bug report or feature request.

Quick recommendations

  • Be precise
  • Be clear: explain 1) how to reproduce the situation, step by step, without leaving any room for interpretation, 2) expected behavior, 3) actual behavior. In separate sections
  • Include only one problem per task
  • Include any relevant links and screenshots

Before you do anything

Can you reproduce the issue?

Try to reproduce your bug to see whether it was just a one-time hiccup.

Has someone else already reported the issue?

You've got a bug or feature request, and you're itching to report it. The first thing you should do is search for an existing report by going to Known Issues. Someone may have already reported the bug or requested the feature.

If you are unsure whether a bug has already been reported, you should report the bug. It is better to have duplicate bugs than it is to have unreported bugs.

Reporting a new bug or feature request

If you have faced a bug and no one else appears to have reported it, then email goglasgow@glasgow.gov.uk and we'll investigate the bug.

That's all! Thanks for your help to improve Glasgow Online!
